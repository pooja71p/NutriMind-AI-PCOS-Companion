[PART 2: ARCHITECTURE & DESIGN] - Duration: 60 seconds
================================================================================

ACTION:
Switch view to show the Agent Workflow Diagram (architecture_diagram.png). Point to the different nodes as you describe them.

SPOKEN NARRATION:
"Let's look under the hood at the system architecture. NutriMind AI is built as a workflow graph using Google's Agent Development Kit, or ADK 2.0.

At the gateway of every interaction is our Security Checkpoint node. This node intercepts all input to perform regex PII scrubbing of email addresses and phone numbers, screens for prompt injections, and logs medical disclaimer warnings.

Once cleared, requests enter the Coordinator Agent. The Coordinator is our central orchestrator, delegating dynamically to seven specialized expert sub-agents:
1. The Profile Agent, which collects and updates basic metrics.
2. The Health Analysis Agent, which calculates daily metabolic stats.
3. The Meal Planner Agent, which constructs daily meal routines.
4. The Lifestyle Agent, providing exercise and sleep guidance.
5. The Progress Agent, which logs progress.
6. The Motivation Agent, offering warm emotional coaching.
7. The Nutrition Knowledge Agent, for answering science-backed questions.

To run intensive calculations and local lookups, these agents connect to a secure local Model Context Protocol, or MCP Server, which runs isolated on stdio."

"Now, let's see a live demonstration of NutriMind AI in action. 

I will enter Anna's details in the playground. As I submit this, the Security Checkpoint logs the audit checklist and passes the cleaned input to the Coordinator. The Coordinator notices that Anna is new and triggers the profile-saving sequence. 

It then delegates to the Health Analysis Agent, which calls the local MCP tool to calculate Anna's BMR, BMI, and a customized PCOS macro split of thirty percent protein, thirty-five percent fats, and thirty-five percent low-GI carbohydrates. 

Following this, it passes the data to the Meal Planner Agent. The meal planner drafts a custom daily routine and calls our local state tool to save the plan under 'latest_meal_plan'. 

Because nutrition is highly subjective, the workflow graph routes to a Human-in-the-Loop approval node, suspending the flow. As you can see, the UI is now prompting us: 'Human Review Required: Do you approve this meal plan?'

I will type 'yes' to approve. The workflow resumes, logs the approved meal plan securely inside the session progress history, and completes with a confirmation message."
